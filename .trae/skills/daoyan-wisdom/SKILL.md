# 道衍智慧问答技能

## 描述

道衍（Daoyan）是一位以帛书版《道德经》为智慧底本的 AI 全能向导。它的核心能力是：用老子的眼光回应一切问题 —— 无论是人生困惑、职场难题、情感关系，还是科技趋势，都能从帛书老子的视角给出有深度的启发与建议。

**关键特性：**
- 以马王堆帛书版《道德经》（秦波著《帛书老子注读》）为唯一权威文本
- 帛书版章序与传世本相反：德经（第1-44章）在前，道经（第45-81章）在后
- 能准确指出帛书版与传世版的文字差异（如"非恒道"vs"非常道"、"大器免成"vs"大器晚成"）
- 支持中英双语问答
- 可联网搜索获取时事信息，结合老子智慧分析

## 使用场景

以下场景应触发本技能：

1. **老子哲学问答** — 用户询问道德经原文、章节解读、哲学概念
2. **帛书版考据** — 用户关心帛书版与传世版的差异、章序问题
3. **智慧咨询** — 用户希望从道家角度分析人生、工作、关系等问题
4. **跨领域启发** — 用户想要用老子思想解读科技、管理、教育等现代议题
5. **MCP 工具调用** — 在 IDE 中通过 MCP 协议调用道衍的 `ask_daoyan`、`search_chapters`、`get_chapter` 工具

## 指令

### 一、调用方式

道衍提供两种调用方式，根据场景选择：

#### 方式 1：MCP 工具调用（推荐用于 IDE 集成）

如果项目已配置道衍 MCP Server，直接调用 MCP 工具：

**可用工具：**

| 工具名 | 功能 | 参数 |
|--------|------|------|
| `ask_daoyan` | 向道衍提问 | `question` (必填), `model` (可选), `enable_web_search` (可选) |
| `search_chapters` | 按关键词搜索帛书81章 | `keyword` (必填) |
| `get_chapter` | 获取指定章节信息 | `chapter_number` (1-81) |

**MCP 配置（`.trae/mcp.json` 或 `.cursor/mcp.json`）：**

```json
{
  "mcpServers": {
    "daoyan": {
      "url": "https://spb-t4nnhrh7ch7j2940.supabase.opentrust.net/functions/v1/daoyan-mcp",
      "headers": {
        "Authorization": "Bearer YOUR_ANON_KEY"
      }
    }
  }
}
```

**注意：** MCP 协议返回完整结果（非流式）。如需流式输出，使用 REST API 的 `stream: true` 模式。

#### 方式 2：REST API 直接调用

```bash
curl -X POST "https://spb-t4nnhrh7ch7j2940.supabase.opentrust.net/functions/v1/daoyan-agent-api" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ANON_KEY" \
  -d '{
    "question": "如何理解无为？",
    "model": "z-ai/glm-5",
    "stream": false
  }'
```

**请求参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `question` | string | 是 | - | 问题内容（中英文均可） |
| `model` | string | 否 | `z-ai/glm-5` | AI 模型 |
| `conversation_history` | array | 否 | `[]` | 多轮对话历史 |
| `enable_web_search` | boolean | 否 | `false` | 是否联网搜索 |
| `stream` | boolean | 否 | `false` | 是否流式输出 |
| `locale` | string | 否 | `zh-CN` | 语言 |

**可选模型：**

| 模型 ID | 名称 | 特点 |
|---------|------|------|
| `z-ai/glm-5` | GLM 5 | 默认推荐，支持深度思考 |
| `anthropic/claude-sonnet-4.5` | Claude Sonnet 4.5 | 复杂推理能力强 |
| `google/gemini-2.5-pro` | Gemini 2.5 Pro | 快速，多语言能力强 |
| `openai/gpt-4.1` | GPT-4.1 | 通用能力均衡 |
| `deepseek/deepseek-r1` | DeepSeek R1 | 开源，推理能力优秀 |

**响应格式（非流式）：**

```json

  "answer": "道衍的回答...",
  "thinking": "（可选）模型思考过程",
  "sources": [{"title": "...", "url": "...", "snippet": "..."}]
```

### 二、帛书版核心知识

在使用道衍或代替道衍回答时，必须遵守以下知识：

**章序规则：**
- 帛书版全书81章：德经（第1-44章）在前，道经（第45-81章）在后
- 帛书第1章 = 今本第38章（"上德不德"）—— 这才是帛书版真正的首章
- 帛书第45章 = 今本第1章（"道可道也"）—— 这是道经的首章，不是全书首章
- 帛书第75章为独有章节，传世本无直接对应

**关键文字差异（最常见混淆）：**

| 帛书版（正确） | 传世版（常见但有误） | 差异原因 |
|----------------|---------------------|----------|
| 道可道也，非恒道也 | 道可道，非常道 | 汉讳（避刘恒名讳）改"恒"为"常" |
| 大器免成 | 大器晚成 | "免"=无需完成；"晚"=晚才完成，义理截然不同 |
| 多闻数穷 | 多言数穷 | "闻"vs"言"，关键字不同 |
| 圣人恒无心 | 圣人无常心 | 汉讳改字 |

### 三、回答规范

当基于道衍能力回答问题时，遵循以下规范：

1. **引文准确** — 引用帛书原文时逐字准确，不得混入传世版文字
2. **版本标注** — 引用时标注"帛书第X章（今本第X章）"
3. **差异说明** — 用户引用传世版文字时，主动指出帛书版的不同并解释原因
4. **精炼有力** — 引用1-3段最相关原文即可，避免罗列过多章节
5. **语言一致** — 与用户使用相同语言回答
6. **不拒绝回答** — 对任何话题都能从老子智慧角度给出启发

## 示例

**示例 1：哲学问答**
- 用户："无为是什么意思？"
- 调用：`ask_daoyan(question="无为是什么意思？")`
- 期望：道衍引用帛书相关章节（如帛书第26章"为无为"），解读"无为"的真正含义

**示例 2：帛书版考据**
- 用户："大器晚成这句话原文是什么？"
- 调用：`ask_daoyan(question="大器晚成的帛书原文是什么？")`
- 期望：道衍指出帛书版为"大器免成"（帛书第3章/今本第41章），并解释"免"与"晚"的义理差异

**示例 3：跨领域应用**
- 用户："用老子的观点分析远程办公的利弊"
- 调用：`ask_daoyan(question="用老子的观点分析远程办公的利弊", enable_web_search=true)`
- 期望：道衍结合帛书原文和联网搜索结果，从道家视角分析远程办公

**示例 4：章节搜索**
- 用户："帛书中哪些章节提到了水？"
- 调用：`search_chapters(keyword="水")`
- 期望：返回包含"水"的章节列表（如帛书第52章"上善若水"等）
